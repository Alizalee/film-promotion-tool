## 一、产品定位

面向影视剧宣发设计师的**本地化拉片工具**，支持自动场景检测、镜头管理、质量评估、视频裁剪导出。所有数据本地处理，不上传任何服务器。

---

## 二、技术架构

| 层 | 技术选型 | 说明 |
|-----|---------|------|
| **后端** | Python FastAPI | RESTful API，uvicorn 运行 |
| **前端** | 单文件 SPA（`static/index.html`） | 内联 HTML + CSS + JS，Apple 风格深色沉浸式界面，**不使用任何前端框架** |
| **图像处理** | OpenCV (DNN + Haar) | 人脸检测、质量评分 |
| **场景检测** | PySceneDetect (ContentDetector) | 自动识别镜头切换点 |
| **视频裁剪** | FFmpeg (subprocess) | 镜头导出和帧提取 |
| **数据存储** | JSON 文件 | 项目索引 + 项目数据，无数据库 |

## 四、数据模型

### 4.1 项目索引 — `workspace/projects.json`

```json
{
  "projects": [ProjectInfo, ...],
  "active_project_id": "proj_d8fa8094" | null
}
```

**ProjectInfo：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一 ID，格式 `proj_{uuid_hex[:8]}` |
| `name` | string | 项目名称，如"白日提灯" |
| `description` | string | 备注说明，可为空 |
| `created_at` | string | 创建时间 `YYYY-MM-DD HH:MM:SS` |
| `updated_at` | string | 最后更新时间 |
| `shot_count` | int | 镜头总数 |
| `video_count` | int | 视频文件数 |

### 4.2 项目数据 — `workspace/projects/{proj_id}/project.json`

```json
{
  "video_path": "/abs/path/to/last_analyzed.mov",
  "video_paths": ["/abs/path/to/video1.mov", "/abs/path/to/video2.mp4"],
  "shots": [Shot, ...],
  "fps": 24.0,
  "total_frames": 1561
}
```

### 4.3 镜头数据 — Shot

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一 ID，格式 `shot_{index:04d}_{video_hash}_{timecode}`，如 `shot_0003_c86d44_00-00-25-04f` |
| `index` | int | 全局序号（追加分析时自动递增偏移） |
| `timecode` | string | 时间码 `HH-MM-SS-FFf` |
| `timecode_display` | string | 显示用 `HH:MM:SS:FF` |
| `start_frame` | int | 起始帧号 |
| `end_frame` | int | 结束帧号 |
| `mid_frame` | int | 中间帧号 = `(start + end) // 2` |
| `start_time` | float | 起始秒数，3 位小数 |
| `end_time` | float | 结束秒数，3 位小数 |
| `duration` | float | 时长秒数，3 位小数 |
| `quality` | float | 质量评分 0-10，1 位小数 |
| `face_count` | int | 人脸数量 |
| `has_person` | bool | `face_count > 0` |
| `favorite` | bool | 是否收藏 |
| `saved` | bool | 是否已保存静帧 |
| `frame_file` | string | 代表帧文件名 `{shot_id}.jpg` |
| `source_video` | string | 源视频绝对路径（区分多视频来源） |

## 五、核心功能模块

### 5.1 多项目管理

- 创建 / 切换 / 重命名 / 删除项目
- 每个项目有独立存储目录和数据
- 顶部栏居中位置放置**项目选择器**：
  - 外观为简洁的文字按钮（当前项目名 + 下拉箭头），点击展开下拉面板
  - 面板采用 `--bg-elevated` 背景 + `--radius-lg` 圆角 + `--shadow-lg` 阴影，宽度 320px
  - 每行显示项目名称 + 次要信息（镜头数 · 视频数），信息文字用 `--text-secondary`
  - 底部一条分割线后放置"新建项目"入口（+ 图标 + 文字按钮风格）
  - 选中项目有 `--accent` 左侧色条标记
- 支持从旧版单项目格式自动迁移到新项目系统

### 5.2 视频导入 ⭐ 【更新】

- **全屏拖拽上传**：
  - 🆕 **任何页面状态下**（无论是空项目还是已有镜头的项目），都支持将视频文件拖拽到浏览器窗口的**任意位置**触发上传
  - 拖入时全屏显示极简遮罩层：`rgba(0,0,0,0.5)` + `backdrop-filter: blur(30px)`，居中一个大号 SF Symbol 风格的下载箭头图标 + 一行提示文字"松开即可导入视频"，文字 `--text-primary`，字号 `--text-xl`
  - 遮罩出现/消失使用 `150ms ease-out` 渐变，不要弹跳动效
  - 不再有虚线边框的 drop zone 区域，保持界面干净
- **空项目状态**：
  - 内容区居中显示极简引导 —— 一个轻量的图标 + 一行说明"拖入视频文件，或点击选择" + 一个主按钮"选择视频文件"
  - 引导区域不加边框和背景色，只用图标 + 文字，保持呼吸感
- **追加分析**：新视频的镜头追加到已有列表（不清除旧数据），支持多视频源管理
- **路径输入**：设置面板中支持手动输入本地视频文件路径进行分析
- 支持 MP4、MOV、AVI 等常见视频格式，支持同时选择多个视频批量分析
- **上传进度**：顶部出现极细的进度条（2px 高，`--accent` 颜色），完成后渐隐消失

### 5.3 自动场景检测与分析

- 使用 PySceneDetect 的 ContentDetector 自动识别镜头切换点
- **灵敏度可调**（10-60，默认 27），数值越小越灵敏，设置面板中以极简滑块呈现
- 每个镜头自动提取中间帧作为代表性静帧（JPEG 95%质量）
- 记录完整的时间码、帧号、时长等元数据
- **分析进度反馈**：
  - 内容区居中显示简洁的圆形进度指示器（Apple 风格 spinner 或百分比环）
  - 下方一行文字说明当前阶段（"正在检测场景…" / "正在提取关键帧…" / "正在分析画质…"）
  - 进度指示使用 `--text-secondary` 颜色，不抢夺注意力

### 5.4 画面质量评分（0-10 分）

**算法公式：**

```
总分 = 清晰度(0~3) + 对比度(0~2.5) + 色彩丰富度(0~2.5) + 亮度合理性(0~2)

1. 清晰度 = min(Laplacian(gray).var() / 500, 1.0) × 3.0
2. 对比度 = min(gray.std() / 128, 1.0) × 2.5
3. 色彩丰富度 = min(HSV_S_mean / 255 × 2, 1.0) × 2.5
4. 亮度合理性 = max(0, 1.0 - |gray_mean/255 - 0.45| × 2) × 2.0

return round(min(total, 10.0), 1)
```

**评分等级**：≥7 优秀 · ≥5 良好 · <5 一般

### 5.5 人脸/人物检测

**双层策略：**

1. **DNN 检测器（优先）**：OpenCV DNN + ResNet-SSD (caffemodel)，置信度阈值 > 0.5
2. **Haar 级联回退**：
   - 正脸（`haarcascade_frontalface_default.xml`，scaleFactor=1.05, minNeighbors=3）
   - 左侧脸（`haarcascade_profileface.xml`）
   - 右侧脸（水平翻转后检测，坐标映射回原图）
   - IoU > 0.35 去重合并
   - **最后手段**：上半身检测器（`haarcascade_upperbody.xml`，minNeighbors=2）

**多帧采样：** 7 个位置 [5%, 15%, 30%, 50%, 70%, 85%, 95%]，取最大人脸数，`max_faces ≥ 2` 时提前退出

### 5.6 镜头浏览与筛选

- **卡片式网格布局**（CSS Grid，最小列宽 240px，间距 16px），卡片风格：
  - 缩略图撑满卡片（16:9 裁切），无内边距
  - 缩略图右上角：质量评分 Badge（小药丸，半透明黑底白字，`--text-xs`）
  - 缩略图左下角：时间码标签（半透明黑底，`--font-mono`，`--text-xs`）
  - 卡片底部信息区：精简为一行 —— 左侧镜头序号，右侧收藏按钮（心形图标，选中为 `--orange`）
  - 卡片无显式边框，`--bg-secondary` 背景，`--radius-md` 圆角
  - hover 态：轻微上浮（translateY -2px）+ 阴影加深，不出现额外边框或覆盖层
  - 选中态（勾选导出时）：卡片右上角出现 `--accent` 圆形勾选标记
- **排序**：工具栏左侧 Segmented Control（分段控件），两个选项"时间" / "质量"，Apple 标准样式
- **筛选**：
  - 工具栏中部放置药丸形筛选标签组，水平排列：
    - "有人物" — 切换 `has_person` 过滤
    - "已收藏" — 切换收藏过滤
    - 🆕 **视频源标签**：当项目包含多个视频时，自动出现视频名称标签（如"视频1"、"视频2"），点击激活筛选，支持多选
  - 评分滑块：最小化设计，只显示当前值 + 滑轨，`--accent` 色滑块，200ms 防抖
  - 筛选标签默认态 `--bg-tertiary`，激活态 `--accent` 背景白字
- **时间码搜索**：工具栏右侧，Apple 标准搜索框样式（放大镜图标 + 输入框），300ms 防抖
- **镜头计数**：搜索框左侧显示"128 个镜头"，`--text-secondary`，不突出

### 5.7 镜头预览弹窗

预览弹窗分为三种内部状态，UI 随状态切换：**播放模式** → **定帧模式** → **裁剪模式**。三种模式共用同一个弹窗容器，不跳转页面。

#### 弹窗容器

- 全屏沉浸式：纯黑遮罩 `rgba(0,0,0,0.85)` + `backdrop-filter: blur(20px)`，视频播放器居中最大化显示
- ESC 键或点击遮罩关闭（裁剪模式下需二次确认）

#### 顶部信息栏（三种模式均显示）

- 浮动半透明栏（`--bg-elevated` + blur），随控制栏联动显示/隐藏
- 左侧：镜头序号 + 时间码（`--font-mono`）
- 右侧操作按钮组（图标按钮）：裁剪编辑、收藏、导出，按钮间距 `--space-3`，每个按钮有 Tooltip
- **定帧模式下**，右侧额外显示**快捷操作条**（见下文），替换原有按钮组

#### 控制栏（播放模式 & 定帧模式）

- 底部浮动半透明栏（`--bg-elevated` + blur），`--radius-lg` 圆角
- 鼠标静止 3 秒后自动淡出，鼠标移动时渐显（Apple TV 风格）
- 控制栏布局（从左到右）：
  - 播放/暂停按钮（空格键触发）
  - 前进/后退 0.5s 按钮
  - **逐帧前进/后退按钮**（键盘 `.` / `,` 触发，Tooltip 标注快捷键）
  - 时间进度条（见下方详细规格）
  - 实时时间码（`--font-mono`，`--text-sm`）

#### 时间进度条（双层设计）

**上层 — 粗定位轨道：**
- 播放模式：高度 2px，`--accent` 色已播区域，hover 时轨道升高至 4px，拖动手柄出现
- 定帧模式（暂停后自动进入）：轨道升高至 6px，强调当前为精确定帧状态，颜色保持 `--accent`

**下层 — 帧缩略图带（定帧模式专属）：**
- 暂停后在进度条下方展开，高度 48px，展示当前镜头连续帧的缩略图序列
- 每帧缩略图宽度自适应（镜头时长短则每帧更宽，时长长则压缩）
- 当前帧位置用 2px 白色竖线标记
- **点击某帧缩略图**：直接跳转到该帧（精确到帧号）
- **鼠标悬停某帧**：上方浮出该帧放大预览（120×68px，`--bg-elevated` 边框，`--radius-sm` 圆角，200ms 延迟出现）
- 帧带展开/收起使用 `200ms ease-out` 动画

#### 定帧模式快捷操作条

暂停后自动进入定帧模式，顶部信息栏右侧按钮组替换为三个快捷按钮（药丸组合，紧凑排列）：

```
[ 复制静帧 ]  [ 设为入点 ]  [ 设为出点 ]
```

- 三个按钮使用次要按钮样式（`--bg-tertiary`），激活态 `--accent`
- 点击"复制静帧"：执行 Canvas → PNG Blob → ClipboardItem，按钮图标短暂变为 ✓（300ms）
- 点击"设为入点"/"设为出点"：自动进入裁剪模式并同步入/出点时间码，省略手动打开裁剪面板步骤
- 恢复播放后快捷操作条消失，还原为原有按钮组

#### 左右镜头导航

- 屏幕两侧大号半透明箭头，默认 opacity 0.3，hover 时 opacity 0.8
- 支持键盘 ← → 切换镜头，切换时画面淡入淡出（200ms）
- 裁剪模式下左右导航箭头隐藏，防止误触

#### 镜头信息面板（按需展开）

- 点击信息按钮，右侧抽屉滑入（300ms Apple 弹性曲线）
- 显示：时长、正片位置、质量评分详情（细长进度条可视化）、人物检测结果

#### 弹窗状态机

```
打开弹窗
  └── 播放模式（默认）
        ├── 按空格/点击暂停 → 定帧模式
        │     ├── 点击帧缩略图 / 按 . / , 逐帧 → 继续定帧模式
        │     ├── 点击"复制静帧" → 执行复制，停留定帧模式
        │     ├── 点击"设为入点/出点" → 进入裁剪模式
        │     └── 按空格恢复播放 → 播放模式
        └── 点击"裁剪编辑"按钮 → 裁剪模式
              ├── 保存裁剪 → 回到播放模式
              └── 取消 → 回到进入前的模式
```

### 5.8 底部全局时间轴

- **全局视频时间轴**（固定底部，Apple Music 迷你播放器风格）：
  - 默认收起为 40px 迷你条，仅显示缩略图条 + 当前镜头位置指示
  - 点击或上拉展开为完整 80px 时间轴
  - 展开态包含：
    - 时间刻度尺（自动适配间隔：5s/10s/30s/1min/5min/10min），刻度文字 `--text-tertiary`，`--text-xs`
    - 所有镜头显示为色块（`--bg-tertiary` 默认，当前镜头 `--accent` 高亮）
    - 播放头：2px 宽竖线 + 顶部小三角，`--text-primary` 白色，实时跟随（50ms 轮询）
    - 裁剪模式下用虚线标注原始范围 vs 裁剪后范围
    - 点击时间轴跳转，拖拽播放头定位
  - 背景 `--bg-secondary` + `backdrop-filter: blur(20px)`
- **缩略图条**：水平可滚动缩略图列表
  - 当前镜头白色 2px 边框标记（不用彩色，保持克制）
  - 当前位置标签（如 "3 / 28"）用小号文字显示在缩略图条右侧
  - 横向滚动平滑，支持触控板手势

### 5.9 镜头裁剪编辑

裁剪模式在预览弹窗内激活，不打开独立弹窗。可通过以下方式进入：
1. 顶部信息栏点击"裁剪编辑"图标按钮
2. 定帧模式下点击快捷操作条的"设为入点"或"设为出点"（自动进入裁剪模式并同步当前帧）

#### 裁剪面板布局

- 从预览弹窗底部滑出的内嵌面板（Apple 标准弹性曲线，300ms）
- 进入裁剪模式时，帧缩略图带保持展开，作为裁剪的视觉参照

**入/出点时间码区（面板顶部）：**

```
入点 [ 00:00:12:08 ]  →  出点 [ 00:00:18:15 ]  |  时长 00:00:06:07
```

- 时间码可直接点击编辑（`--font-mono`，focus 时 `--accent` 边框）
- 入/出点各有一个对应的**帧缩略图预览**（80×45px，`--bg-elevated` 边框，`--radius-sm`），实时反映当前入/出点所在帧，让用户"看着帧"调整而非"看着数字"

**微调按钮组（两行，入点和出点各一行）：**

```
入点：  [ ←1帧 ]  [ ←0.1s ]  [ ←0.5s ]  [ →0.5s ]  [ →0.1s ]  [ →1帧 ]
出点：  [ ←1帧 ]  [ ←0.1s ]  [ ←0.5s ]  [ →0.5s ]  [ →0.1s ]  [ →1帧 ]
```

- "1帧"按钮精确到 `1/fps` 秒，解决 ±0.1s 不够精准的问题
- 使用次要按钮样式，紧凑排列，按钮高度 28px

**快捷定位（文字按钮风格）：**
- "播放头 → 入点" / "播放头 → 出点"：将当前播放位置直接设为入/出点
- "重置"：恢复原始镜头范围

#### 进度条裁剪手柄

- 帧缩略图带上叠加两个可拖拽的竖线手柄（入点 `--accent` 蓝色，出点 `--green` 绿色），2px 宽 + 顶部小三角
- 入/出点之间区域轻微高亮（`--accent` 10% 透明度背景）
- 手柄拖动时对应时间码和帧缩略图预览实时同步更新

#### 操作栏（面板底部）

- 左侧"取消"（次要按钮），右侧"保存裁剪"（主按钮）
- 保存后显示简洁 Toast 确认，裁剪面板收起，回到播放模式

- **降级下载**：Clipboard API 不可用时自动下载 PNG
- 操作反馈：按钮图标短暂切换为"✓"（300ms），无弹窗干扰

### 5.11 收藏功能

- 支持"只看收藏"筛选

### 5.12 镜头合并 ⭐ 【新增功能】

支持将相邻或非相邻的两个镜头合并为一个镜头，合并后的镜头时间范围 = 较早镜头的入点到较晚镜头的出点，中间帧自动重新提取。

#### 触发方式

**方式一：拖拽合并（主要入口）**
- 在镜头网格列表中，长按（300ms）某个镜头卡片进入"拖拽模式"
  - 被拖拽卡片轻微放大（scale 1.05）+ 阴影加深，`--accent` 边框出现
  - 其他卡片进入"可接收"状态：`--bg-tertiary` 背景加深，等待放置
- 拖拽移动到目标卡片上方时，目标卡片出现 `--accent` 高亮边框 + 居中显示"合并"文字提示（`--text-sm`，半透明黑底）
- 松开鼠标 → 弹出合并确认弹窗（见下方）

**方式二：预览弹窗内合并（精细操作入口）**
- 顶部信息栏新增"合并"图标按钮（两个矩形合一的图标）
- 点击后顶部出现选择提示："请在下方选择要合并的另一个镜头"，缩略图条进入可点击选择状态
- 点击缩略图条中的目标镜头 → 弹出合并确认弹窗

#### 合并确认弹窗

- Apple 标准居中弹窗，`--bg-elevated` 背景，`--radius-lg` 圆角
- 弹窗内并排展示两个镜头的代表帧缩略图（各 120×68px）+ 箭头图标居中
- 说明文字（`--text-secondary`）：
  - 合并后时间范围（如 `00:00:12:00 → 00:00:25:08`）
  - 合并后时长
  - 如果两镜头不相邻：显示警告"两个镜头之间存在间隔，合并后将包含中间片段"
- 底部：左"取消"（次要按钮），右"合并"（主按钮）

#### 合并逻辑（后端）

- 合并后新镜头的元数据：
  - `start_frame` = min(shot_a.start_frame, shot_b.start_frame)
  - `end_frame` = max(shot_a.end_frame, shot_b.end_frame)
  - `mid_frame` 重新计算，并重新提取代表帧 JPEG
  - `quality` / `face_count` 取两者中较高值
  - `source_video` 必须相同（不同视频来源的镜头不允许合并，弹窗阶段提前拦截并提示）
  - `id` 生成新 ID，原两个 Shot 的 ID 废弃
- 原两个镜头从列表中删除，新合并镜头插入到原较早镜头的位置
- `index` 全局重排，保证连续性

#### 约束与错误处理

| 场景 | 处理方式 |
|------|---------|
| 两个镜头来自不同视频源 | 拖拽放置时目标卡片不高亮，显示禁止图标；确认弹窗阶段拦截并提示"不同视频来源的镜头无法合并" |
| 合并后时长超过原始视频范围 | 后端校验，返回错误，Toast 提示 |
| 同一镜头拖到自身 | 忽略操作，不触发合并流程 |

### 5.13 批量选择与导出

## 九、UI/UX 设计系统 — Apple 风格

> **设计理念**：参考 Apple Human Interface Guidelines，追求**简洁克制、内容优先、直觉交互**。减少视觉噪音，让用户专注于画面内容本身。

### 9.1 设计原则

| 原则 | 说明 |
|------|------|
| **内容优先** | UI 元素退后，视频画面和镜头缩略图是绝对主角，控件只在需要时出现 |
| **减法设计** | 去掉一切非必要的装饰、边框和色块，用留白和间距建立层次 |
| **一致性** | 全局统一的圆角、间距、动效节奏，形成可预测的操作体验 |
| **渐进式呈现** | 不一次展示所有功能，根据上下文渐进显示操作项（如 hover 才出现操作按钮） |
| **无障碍** | 对比度符合 WCAG AA 标准，可聚焦元素有清晰的 focus 态 |

### 9.2 色彩系统

采用**深色模式**为主色调，致敬 Final Cut Pro / Apple TV 的沉浸式暗色界面：

```css
:root {
  /* 背景层级（由深到浅，层级越高越浅） */
  --bg-primary:    #000000;       /* 纯黑底，最大化画面对比度 */
  --bg-secondary:  #1c1c1e;       /* 卡片、面板背景 — Apple 标准暗色 */
  --bg-tertiary:   #2c2c2e;       /* 输入框、hover 态 */
  --bg-elevated:   #3a3a3c;       /* 弹窗、浮层 */

  /* 文字 */
  --text-primary:   #f5f5f7;      /* 主文字 — Apple 标准白 */
  --text-secondary: #a1a1a6;      /* 次要信息、说明文字 */
  --text-tertiary:  #636366;      /* 占位符、禁用态 */

  /* 强调色（克制使用，仅用于关键操作和状态） */
  --accent:         #0a84ff;      /* Apple Blue — 主操作按钮、选中态 */
  --accent-hover:   #409cff;      /* 悬浮态 */
  --green:          #30d158;      /* 成功、当前选中镜头 */
  --orange:         #ff9f0a;      /* 警告、收藏星标 */
  --red:            #ff453a;      /* 危险操作、删除 */

  /* 分割线 */
  --separator:      rgba(84, 84, 88, 0.65);  /* Apple 标准分割线 */
}
```

> **色彩使用原则**：强调色在画面中占比不超过 10%，大面积使用中性深色，让内容（视频帧）自然成为视觉焦点。

### 9.5 动效

| 场景 | 动效 | 参数 |
|------|------|------|
| **常规过渡** | 所有状态变化 | `200ms ease-out` |
| **弹窗出现** | 从下方滑入 + 渐显 | `300ms cubic-bezier(0.32, 0.72, 0, 1)` — Apple 标准弹性曲线 |

> **动效原则**：动效服务于功能反馈，不是装饰。时长控制在 150-300ms，保持轻快。使用 Apple 标准的弹性贝塞尔曲线。

## 十、Token 优化与版本管理策略 ⭐

### 10.1 代码拆分降低 Token 消耗

当前最大的问题是 `index.html` 单文件 2530 行（~90KB），每次让 AI 修改都需要传输全量文件，消耗大量 token。

**建议拆分方案：**

```
static/
├── index.html         # 纯 HTML 骨架（~200 行）
├── css/
│   ├── base.css       # 主题变量、reset、通用样式
│   ├── layout.css     # header、toolbar、grid 布局
│   ├── components.css # 卡片、弹窗、按钮组件
│   └── timeline.css   # 底部时间轴、缩略图条
└── js/
    ├── state.js       # 全局状态变量定义
    ├── api.js         # fetch 封装和所有 API 调用
    ├── projects.js    # 项目管理（创建/切换/删除）
    ├── upload.js      # 视频上传和拖拽处理
    ├── shots.js       # 镜头列表加载、渲染、筛选
    ├── preview.js     # 预览弹窗、播放控制、导航
    ├── trim.js        # 裁剪编辑器
    ├── timeline.js    # 全局时间轴、缩略图条
    ├── export.js      # 导出和文件夹浏览
    └── utils.js       # 工具函数（时间格式化、Toast等）
```

**好处：**
- 修改某个功能时只需要传递相关的 1-2 个文件（~100-300 行），而不是整个 2530 行
- 预估 **token 消耗降低 70-80%**
- 更好的版本控制，git diff 更清晰

### 10.2 后端代码拆分

当前 `app.py` 1222 行，同样建议拆分：

```
app.py                    # 主入口，仅包含 FastAPI app 创建和路由注册（~50 行）
routes/
├── projects.py           # 项目管理 API
├── videos.py             # 视频上传/分析/管理 API
├── shots.py              # 镜头数据 API
├── files.py              # 文件服务 API（视频流、帧图片）
└── export.py             # 导出 API
services/
├── scene_detect.py       # 场景检测核心逻辑
├── quality.py            # 质量评分算法
├── face_detect.py        # 人脸检测（DNN + Haar）
└── project_manager.py    # 项目数据管理（load/save/migrate）
models/
├── schemas.py            # Pydantic 请求模型
└── constants.py          # 配置常量（路径、阈值等）
```

### 10.3 增量修改策略

当需要让 AI 修改代码时：
1. **只发送需要修改的文件**，不发送全量代码
2. **使用模块引用**：如"修改 `js/preview.js` 中的 `openPreview` 函数"
3. **每次修改后提交 git**，方便回滚
4. **维护这份规格书**：功能变更先更新规格书，再让 AI 基于规格书修改代码

### 10.4 版本管理建议

```bash
# 初始化 git（如果还没有）
git init
git add -A
git commit -m "v2.0 代码拆分前备份"

# 每次大的功能迭代后
git add -A
git commit -m "feat: 添加全屏拖拽上传功能"
```

| 问题 | 说明 | 优先级 |
|------|------|--------|
| DNN 模型每帧重新加载 | `detect_faces()` 每次调用都 `readNetFromCaffe()`，应全局加载一次 | 高 |
