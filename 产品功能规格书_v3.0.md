## 1. 产品定位与概述

### 1.1 产品名称

**Nice Cut** — 影视剧宣发拉片工具

### 1.2 目标用户

影视剧宣发设计师、物料制作人员。用于从影视剧正片中快速拆分镜头、筛选人像素材、裁剪导出所需片段。

### 1.3 核心价值

- **自动拉片**：上传视频后自动识别场景切换，拆分为独立镜头
- **智能分类**：基于人脸检测自动将镜头分为近景人像/黄金人像/远景人像/空镜四类
- **快速筛选**：支持按镜头类型、来源视频、收藏状态、人数等多维度筛选
- **精准裁剪**：在预览窗口中进行二次精修（Trim / Split），精确到帧
- **批量导出**：一键导出选中镜头到本地目录或浏览器下载

### 1.4 产品形态

纯本地运行的 Web 应用。后端 FastAPI 服务 + 前端纯 HTML/CSS/JS 单页应用，通过浏览器访问 `http://localhost:12345`。不依赖任何外部服务或数据库，数据以 JSON 文件持久化。

## 5. 核心算法与检测机制

### 5.1 场景检测（Scene Detection）

- **算法**：PySceneDetect 的 `AdaptiveDetector`
- **默认阈值**：`DEFAULT_THRESHOLD = 27.0`（用户可调，范围 1-100）
- **边界修剪**：`BOUNDARY_TRIM_FRAMES = 2`，每个镜头首尾各裁去 2 帧，防止跨镜头帧泄漏
- **流程**：
  1. 使用 AdaptiveDetector 检测场景切换点
  2. 根据切换点拆分为镜头列表
  3. 每个镜头提取封面帧（中间帧），同时生成缩略图和全尺寸帧
  4. 缩略图：480px 宽，JPEG 80% 质量
  5. 全尺寸帧：原始分辨率，JPEG 95% 质量

### 5.2 两阶段分析

分析过程分为两个阶段：

**阶段一：快速拆分（同步，前台）**
- 执行场景检测，拆分镜头
- 提取封面帧，生成缩略图
- 完成后立即返回镜头列表，用户可开始浏览

**阶段二：深度分析（异步，后台线程）**
- 对每个镜头执行人脸检测
- 计算运动评分（Motion Score）
- 执行镜头分类（Shot Type Classification）
- 通过进度轮询接口实时反馈进度
- 支持中途取消和断点续做

### 5.3 人脸检测（Face Detection）

- **模型**：OpenCV 内置 YuNet（`face_detection_yunet_2023mar.onnx`）
- **全局单例**：模型只加载一次，全局复用
- **中文路径兼容**：如果模型路径包含非 ASCII 字符，自动复制到临时目录加载
- **检测参数**：
  - 置信度阈值：`YUNET_SCORE_THRESHOLD = 0.6`
  - NMS 阈值：`YUNET_NMS_THRESHOLD = 0.3`
  - Top-K：`YUNET_TOP_K = 20`
- **多帧检测策略**：
  - 从镜头中均匀采样多帧（默认 5 帧）
  - 对每帧分别执行人脸检测
  - 选择"最佳帧"（人脸最大、构图最优）作为封面帧
  - 最终结果取最佳帧的检测数据

### 5.4 有效区域检测（Effective Region Detection）

- **目的**：排除视频中的黑边区域，只在有效画面内计算人脸比例
- **方法**：将帧转为灰度图，按行/列求均值，找到亮度高于阈值的区域
- **黑边阈值**：`BLACK_BAR_THRESHOLD = 25`
- **最小有效比例**：`MIN_EFFECTIVE_RATIO = 0.3`（如果检测到的有效区域过小则使用全帧）
- **缓存机制**：同一视频的有效区域只计算一次，后续复用

### 5.5 人脸构图分析（Face Composition）

对检测到人脸的帧进行构图质量评估：

| 分析项 | 说明 |
|--------|------|
| **裁切检测（face_cropped）** | 人脸框是否超出画面边界（含 `CROP_MARGIN = 0.02` 容差） |
| **安全区域检测（face_in_safe_zone）** | 人脸中心是否在画面中心 80% 区域内（`SAFE_ZONE_MARGIN = 0.1`） |
| **头顶留白（head_margin_ratio）** | 人脸框顶部到画面顶部的距离比例 |

### 5.6 四级镜头分类（Shot Type Classification）

基于 `face_ratio`（人脸面积占有效区域比例）和构图安全性进行分类：

| 类型 | 条件 |
|------|------|
| **近景人像** | `face_ratio ≥ FACE_RATIO_TIER_HIGH (0.07)` |
| **黄金人像** | `FACE_RATIO_TIER_LOW (0.007) ≤ face_ratio < 0.07` 且 `face_in_safe_zone = true` 且 `face_cropped = false` |
| **远景人像** | `face_ratio ≥ MIN_FACE_RATIO (0.002)` 但不满足以上条件 |
| **空镜** | `face_ratio < MIN_FACE_RATIO (0.002)` 或未检测到人脸 |

### 5.7 运动评分（Motion Score）

- **算法**：Lab 色彩空间帧差法
- **流程**：
  1. 从镜头中均匀采样帧
  2. 将相邻帧转换到 Lab 色彩空间
  3. 计算相邻帧的像素差异均值
  4. 归一化到 0-100 分
- **用途**：辅助筛选静态/动态镜头

1. 如果镜头有 `clip_file` 且文件存在 → 直接服务裁剪文件（完整文件）
2. 否则 → 服务源视频文件（浏览器通过 Range 请求按需加载）
3. 支持缓存版本控制（通过 `v` 查询参数避免浏览器缓存过期文件）

#### FFmpeg 裁剪策略

- **双 -ss 策略**：在输入和输出端都使用 `-ss` 参数，兼顾定位速度和精度
- **自动发现 FFmpeg**：在 Windows 上自动搜索系统 PATH 和常见安装目录
- **命令模板**：
  ```
  ffmpeg -ss {start} -i {input} -ss 0 -t {duration} -c:v libx264 -c:a aac -y {output}
  ```

### 7.3 镜头卡片（Shot Card）

每张镜头卡片展示：
- **缩略图**：480px 宽，高斯模糊背景 + `object-fit: contain` 前景（解决宽高比不一致的问题）
- **悬停视频预览**：鼠标悬停 300ms 后自动播放预览视频（2 倍速）
- **时长标签**：右下角显示镜头时长
- **镜头类型标签**：左上角显示类型（带颜色区分）
- **收藏标记**：星标图标
- **来源标记**：孤儿镜头显示特殊标记（源视频已删除）

### 7.4 悬停视频预览

- 鼠标进入卡片后延迟触发（防抖）
- 创建 `<video>` 元素叠加在缩略图上
- 播放速率 2x
- 鼠标离开立即销毁视频元素
- 优先使用 clip_file（已裁剪文件），否则使用源视频的时间范围

### 7.5 预览模态框（Preview Modal）

**常规模式**：
- 全屏视频播放，带播放/暂停、进度条
- 播放速率控制（0.25x / 0.5x / 1x / 1.5x / 2x）
- 左右箭头在同源视频的镜头间导航
- 键盘快捷键：空格暂停、左右方向键跳帧、Esc 关闭
- 复制当前帧到剪贴板
- 一键导出当前镜头

**编辑模式（Edit Mode）**：
- **Trim**：拖拽调整镜头的起止时间，实时预览
- **Split**：在当前播放位置拆分镜头
- 精确到帧的边界监控（requestAnimationFrame 循环）
- 确认/取消操作

**调试面板（Debug Panel）**：
- 三击视频区域触发显示/隐藏
- 显示镜头原始数据（JSON 格式）
- 包括所有检测参数：face_ratio、motion_score、head_margin_ratio 等

### 7.6 拖拽合并（Drag Merge）

- 长按镜头卡片 300ms 触发拖拽模式
- 拖拽到目标卡片上松开，合并为一个镜头
- 只能合并同一源视频的镜头
- 合并后镜头时间范围取两者的并集
- 视觉反馈：拖拽时显示半透明覆盖层、目标高亮

### 7.7 多选与批量操作

- 点击选择栏进入多选模式
- 点击卡片切换选中状态
- 批量操作按钮：
  - **批量收藏 / 取消收藏**
  - **批量删除**
  - **批量导出**
- 选择栏显示已选数量

### 7.8 上传交互

三种上传场景：

| 场景 | 触发条件 | 行为 |
|------|----------|------|
| 空项目单视频 | 项目无视频，上传 1 个文件 | 同步快速分析 → 直接展示 |
| 空项目多视频 | 项目无视频，上传多个文件 | 第一个同步分析 → 其余后台批量分析 |
| 已有视频追加 | 项目已有视频，上传新文件 | 后台追加分析，不中断当前浏览 |

- 支持拖拽上传和点击选择
- 上传前自动检查重复文件名
- 支持的视频格式：`mp4, avi, mkv, mov, wmv, flv, webm, m4v, ts, mts, m2ts, mpg, mpeg, vob, 3gp, rm, rmvb`

## 8. 后台任务系统

### 8.1 架构

- 使用 Python `threading` 实现后台分析
- 全局单任务：同一时间只有一个后台分析任务在运行
- 前端通过轮询 `GET /bg_task_status` 获取进度

### 8.2 任务类型

| 类型 | 触发 | 说明 |
|------|------|------|
| `deep_analysis` | 单视频快速分析后自动启动 | 对已拆分镜头执行人脸检测 + 分类 + 运动评分 |
| `batch_analysis` | 批量上传多视频 | 依次对多个视频执行完整分析 |
| `reanalyze` | 用户修改阈值重新分析 | 以新阈值重新拆分镜头并深度分析 |

### 8.3 取消机制

- 设置全局取消标志位
- 后台线程在每个镜头处理循环中检查标志位
- 检测到取消后优雅退出，保留已完成的分析结果

### 8.4 断点续做（Resume）

- `GET /analysis_completeness` 检查是否存在未完成深度分析的镜头（`shot_type` 为空）
- `POST /resume_analysis` 仅对未完成的镜头执行深度分析
- 适用于分析中途被中断（如取消、关闭浏览器）后恢复

### 8.5 进度反馈

```json
{
  "running": true,
  "progress": 65,          // 百分比
  "current_shot": 13,      // 当前处理的镜头序号
  "total_shots": 20,       // 总镜头数
  "task_type": "deep_analysis",
  "video_name": "example.mp4"
}
```

前端轮询间隔：约 1-2 秒，通过 `setInterval` 实现。

## 9. 特色机制详解

### 9.1 收藏持久化（Favorites Persistence）

**问题**：重新分析视频时，镜头会被重新生成（新 UUID），导致基于 ID 的收藏失效。

**解决方案**：
- 收藏信息独立存储在 `project.json` 的 `favorites` 数组中
- 每条收藏记录保存 `source_video`、`start_time`、`end_time`
- 加载镜头时，通过时间范围匹配（容差匹配）动态恢复 `is_favorite` 状态
- 重新分析后，新镜头自动继承匹配的收藏状态

### 9.2 孤儿镜头系统（Orphan Shots）

**场景**：用户删除源视频，但希望保留已收藏的精选镜头。

**流程**：
1. 删除视频时，用户可选择 `keep_favorites = true`
2. 系统找到该视频所有收藏镜头
3. 使用 FFmpeg 将每个收藏镜头预裁剪为独立 MP4 文件（存入 `clips/` 目录）
4. 修改这些镜头的 `source_video` 为 `"__orphan__"`
5. 删除源视频文件和非收藏镜头
6. 孤儿镜头可正常播放（使用自己的 clip_file）、导出
7. 前端以特殊标记区分孤儿镜头

### 9.3 智能封面帧选择

当执行人脸检测时（多帧采样），系统会自动更新封面帧：
- 从采样帧中选择人脸面积最大、构图最佳的帧
- 重新生成该帧的缩略图和全尺寸图
- 确保镜头卡片展示的是最有代表性的画面

### 9.4 收藏自动裁剪

- 用户收藏一个镜头时，后台自动调用 FFmpeg 将该时间段裁剪为独立 MP4 文件
- 裁剪文件存储在 `clips/` 目录
- 后续播放预览时直接使用裁剪文件，加载更快
- 为孤儿镜头机制提供基础（源视频删除后仍可播放）

### 9.5 中文路径兼容

- YuNet 模型加载时，如果路径包含非 ASCII 字符，OpenCV 会加载失败
- 系统自动检测路径，必要时将模型文件复制到系统临时目录（ASCII 路径）
- 对用户完全透明，无需手动处理
