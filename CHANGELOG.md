# CHANGELOG — Nice Cut 影视剧宣发拉片工具

> 记录 v3.0 文档版本以来新增的功能点和重要改动。

---

## 2026-05-11 — 性能优化与代码重构

### 性能优化
- **project_manager.py 内存缓存**：基于 mtime 的项目数据缓存，消除重复 JSON IO
- **favorites 匹配算法优化**：按 source_video 分组索引，复杂度从 O(n×m) 降为 O(n×k)
- **后端 /api/shots 增加 per_video_counts**：一次性返回按视频源分组的计数，前端不再额外请求
- **前端 loadShots() 优化**：消除视频源筛选时的 2~3 次额外 API 请求

### 代码重构
- **新增 services/clip_service.py**：统一 FFmpeg 裁剪逻辑（双 -ss 策略），routes/ 中 4 处重复裁剪代码改为调用此服务
- **新增 services/shot_enrichment.py**：统一人脸检测 + 构图标记 + 景别分类逻辑，routes/shots.py 中 4 处重复代码改为调用
- **新增 services/bg_task.py**：从 routes/videos.py 提取后台任务管理（线程控制、取消机制、状态字典）
- **更新端口说明**：实际端口为 8000（非文档中的 12345）

---

## 2026-05-09 以前 — v3.0 后的增量功能

### 新增功能
- **人数筛选**：支持按 1 人 / 2 人 / ≥3 人单选过滤镜头（前端 `peopleFilter`）
- **拖拽合并镜头**：长按卡片进入拖拽，释放到同源镜头可合并为一个大镜头
- **拆分镜头**：在预览界面指定时间点拆分为前后两段（`/api/split_shot`）
- **视频重命名**：双击侧边栏视频名内联编辑，调用 `/api/videos/rename`
- **批量删除镜头**：选择模式下批量选中后可一键删除（`/api/shots/delete`）
- **批量收藏/取消收藏**：选择模式下一键全部收藏或全部取消收藏
- **孤儿镜头（其他片段）**：删除源视频时保留收藏镜头，source_video 标记为 `__orphan__`
- **收藏自动裁剪**：收藏时自动预裁剪独立 MP4，失去源视频后仍可播放
- **补偿裁剪接口**：`/api/ensure_favorite_clips` 扫描缺失 clip 的收藏并补裁
- **景别筛选空镜时隐藏人数选择器**

### 改进
- **多帧采样人脸检测**：由单帧改为 25%/50%/75% 三帧采样，准确率提升
- **最优帧封面选取**：根据人脸质量选取最佳帧作为镜头封面
- **构图瑕疵标记**：裁头/贴边/黑边检测并在前端展示标签
- **有效区域缓存**：视频黑边检测结果缓存，避免重复计算
- **选中操作栏**：底部浮动栏显示已选缩略图和快捷操作按钮
- **视频源 Tooltip**：侧边栏视频项 hover 显示时长、镜头数、文件大小

### API 说明
- 所有 API 路径均以 `/api` 前缀开头（如 `/api/shots`、`/api/videos`）
- 实际服务端口为 `8000`（通过 uvicorn 启动）
